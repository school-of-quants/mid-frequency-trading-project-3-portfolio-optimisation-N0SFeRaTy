# Обзор стратегии

## 1) Данные

- К данным применяется **маска членства**: OHLCV выставляются в `NaN` на датах, когда тикера не было в индексе.
- Тикеры с недостатком данных исключаются
- Отчёт качества скачивания: `equity_project/data/pony/data_quality_report.json`.

## 2) Фичи 

Фичи:
- доходности на разных горизонтах (`ret1/5/10/22/63/126/252`),
- ранги momentum (`mom*_rank`) и ранги волатильности,
- relative returns к прокси рынка (equal-weight),
- отклонения от скользящих средних (dev5/22/63/252, ma200vs50),
- волатильность (`vol5/22/63/252`), Sharpe-like (`sharpe22/63`), beta63,
- intraday/range (`open_to_close`, `high_low_range`),
- liquidity/volume (`log_dollar_volume`, `dollar_volume_rank`, `volume_change22`).


## 3) Метки (labels) — Triple Barrier

Идея:
- Для каждой даты `t` и тикера строится траектория `Close[t : t+horizon]`.
- Горизонт (вертикальный барьер) задаётся в **торговых днях**: `LABEL_HORIZON_DAYS = 10`.
- Горизонтальные барьеры зависят от волатильности:
	- считается rolling std 1-дневных доходностей,
	- `trgt[t] = clip(scaling_factor * rolling_std, min_trgt, max_trgt)`

Параметры:
- `LABEL_HORIZON_DAYS = 10`
- `LABEL_ROLLING_N = 50`
- `LABEL_SCALING_FACTOR = 1.5`
- `LABEL_MIN_TRGT = 0.008`
- `LABEL_MAX_TRGT = 0.06`
- `LABEL_NEUTRAL_THRESHOLD_MULT = 0.25`

Правило присвоения класса:
- Если верхний барьер достигнут раньше нижнего → метка `+1`.
- Если нижний раньше верхнего → метка `-1`.
- Если ни один барьер не достигнут до горизонта → метка определяется по знаку доходности на горизонте, с нейтральной зоной:
	- `0`, если $|r_{t\to t+h}| < neutral_threshold_mult \cdot trgt[t]$,
	- иначе `sign(r_{t\to t+h})`.

Пропуски:
- Если окно будущих цен неполное/содержит NaN или `t` близко к концу ряда (нет полного горизонта), метка становится `NaN` и затем отбрасывается

## 4) Обучение модели

- Модель: `CatBoostClassifier` для многоклассовой классификации по меткам `{-1, 0, 1}`.
- Валидация: `PurgedKFold` 
	- purge/embargo выставлены равными горизонту метки (`LABEL_HORIZON_DAYS = 10`).
- Метрики, которые сохраняются:
	- `accuracy`, `balanced_accuracy`, `macro_f1` по фолдам и в OOF.

## 5) Портфельная конструкция и бэктест

### 5.1 Сигнал
- На backtest периоде считаются `predict_proba`.
- Базовый score:
	- `score = P(class=+1) - P(class=-1)`
- Дальше:
	- отсекаются слабые сигналы: `score >= MIN_SCORE` (иначе 0),
	- берутся top-N по score: `TOP_N = 35`,
	- long-only: отрицательные и нулевые выкидываются.

- Чтобы не сидеть долго без позиции при слабых сигналах:
- Если активных сигналов меньше `MIN_POSITIONS = 10`, стратегия переходит в диверсифицированный fallback (все веса равные до risk-adjustment).

### 5.2 Risk-adjustment и ограничения
- Risk-adjustment: масштабирование на inverse-vol по фиче `VOL_FEATURE_NAME = "vol22"`.
- Ограничение концентрации: `MAX_WEIGHT = 0.07` на бумагу.

### 5.3 Ребаланс
- Ребаланс раз в `REBALANCE_EVERY_N_DAYS = 5` торговых дней.
- На днях без ребаланса таргеты `NaN` чтобы постоянно не ребаланситься и не тратить кучу денег на комиссии
- Снижение оборота (чтобы не сжигать доходность комиссиями):
	- Плавный ребаланс: в день ребаланса мы не пересобираем портфель “с нуля”, а смешиваем новые целевые веса со старыми. При `REBALANCE_BLEND_ALPHA = 0.3` портфель сдвигается на 30% к новым весам и на 70% остаётся как был — так сделок меньше.
	- Отсечение хвоста мелких позиций: после смешивания часто появляются десятки/сотни очень маленьких позиций. В день ребаланса оставляем только `TOP_N` самых крупных позиций по весу, а остальные обнуляем — чтобы портфель не расползался в “мусорные” микропозиции и не создавал лишние сделки.
